# Family Drama — About Page Build Instructions

## Project Overview

Build a premium production company About page for **Family Drama** (rebrand of Bolt TV Studios). Family Drama is a full service production company specializing in commercials, branded content, documentaries, special projects, and director driven storytelling.

This page uses a **horizontal scroll paradigm on desktop** (≥1025px) and collapses to **vertical scroll on mobile** (≤1024px). The design is bold, cinematic, and minimal. All visual richness comes from photography and video content, not UI decoration. The page tells a narrative story across 15 slides, building from identity to philosophy to proof to a project showcase finale.

**Reference sites for feel and quality bar:**
- https://prettybird.co/about/ (primary reference, same horizontal scroll paradigm)
- https://toolofna.com/about/ (secondary reference, animation quality)

**Tech stack:** React (Vite), CSS Modules or styled-components, no external animation libraries (no GSAP, no Lenis, no Framer Motion). All animations are CSS transitions triggered by Intersection Observer via JavaScript. This vanilla approach is intentional and non-negotiable.

## 1. Architecture & Scroll System

### Horizontal Scroll (Desktop ≥ 1025px)

The entire About page is a single horizontal scroll container. The user scrolls vertically with their mouse wheel or trackpad, but the page moves horizontally. This is achieved by translating vertical scroll input into horizontal position via JavaScript.

**DOM Structure:**

```html
<div class="page-about">
  <div class="page-about__wrapper">
    <div class="slide slide--1 slide--hero">...</div>
    <div class="slide slide--2 slide--difference">...</div>
    <div class="slide slide--3 slide--philosophy">...</div>
    <!-- ... slides 4 through 14 ... -->
    <div class="slide slide--15 slide--showcase">
      <!-- This slide contains nested VERTICAL scroll -->
      <div class="showcase__vertical-container">
        <div class="showcase__card showcase__card--1">...</div>
        <div class="showcase__card showcase__card--2">...</div>
        <div class="showcase__card showcase__card--3">...</div>
        <div class="showcase__card showcase__card--4">...</div>
        <div class="showcase__card showcase__card--5">...</div>
      </div>
    </div>
  </div>
</div>
```

**CSS for horizontal layout:**

```css
.page-about {
  width: 100vw;
  height: 100vh;
  overflow: hidden;
  position: fixed;
  top: 0;
  left: 0;
}

.page-about__wrapper {
  display: flex;
  flex-direction: row;
  width: max-content;
  height: 100vh;
  will-change: transform;
}

.slide {
  width: 100vw;
  height: 100vh;
  flex-shrink: 0;
  position: relative;
  overflow: hidden;
}
```

**JavaScript scroll translation:**

```javascript
// Core horizontal scroll logic
class HorizontalScroller {
  constructor() {
    this.wrapper = document.querySelector('.page-about__wrapper');
    this.currentX = 0;
    this.targetX = 0;
    this.maxScroll = 0;
    this.ease = 0.08; // Smooth interpolation factor
    this.isShowcaseActive = false;

    this.init();
  }

  init() {
    this.calculateMaxScroll();
    window.addEventListener('wheel', this.onWheel.bind(this), { passive: false });
    window.addEventListener('resize', this.calculateMaxScroll.bind(this));
    this.animate();
  }

  calculateMaxScroll() {
    // Total scrollable width minus one viewport
    this.maxScroll = this.wrapper.scrollWidth - window.innerWidth;
  }

  onWheel(e) {
    e.preventDefault();

    // Check if we're in the showcase slide (slide 15) with nested vertical scroll
    if (this.isShowcaseActive) {
      // Let vertical scroll happen inside the showcase container
      return;
    }

    // Translate vertical delta to horizontal movement
    const delta = e.deltaY || e.deltaX;
    this.targetX = Math.max(0, Math.min(this.maxScroll, this.targetX + delta));
  }

  animate() {
    // Smooth interpolation between current and target
    this.currentX += (this.targetX - this.currentX) * this.ease;
    this.wrapper.style.transform = `translateX(${-this.currentX}px)`;

    // Dispatch custom event for parallax and intersection calculations
    window.dispatchEvent(new CustomEvent('horizontalscroll', {
      detail: { scrollX: this.currentX, progress: this.currentX / this.maxScroll }
    }));

    requestAnimationFrame(this.animate.bind(this));
  }
}
```

**Critical details:**
- The native scrollbar must be completely hidden. Use `scrollbar-width: none` and `::-webkit-scrollbar { display: none }`.
- The `ease` value of 0.08 creates a buttery smooth scroll feel. Too high and it feels snappy. Too low and it feels laggy.
- Touch events on mobile bypass this entirely (mobile uses native vertical scroll).
- The `will-change: transform` on the wrapper enables GPU compositing.

### Nested Vertical Scroll (Slide 15 only)

Slide 15 contains a vertically scrolling container inside the horizontal page. When the user scrolls to slide 15, horizontal scrolling stops and vertical scrolling takes over inside the showcase container.

```css
.slide--showcase {
  width: 100vw;
  height: 100vh;
  overflow: hidden;
}

.showcase__vertical-container {
  width: 100%;
  height: 500vh; /* 5 cards × 100vh each */
  position: relative;
}

.showcase__card {
  width: 100vw;
  height: 100vh;
  position: relative;
  overflow: hidden;
}
```

**Detection logic:** When `scrollX` reaches the position of slide 15, set `isShowcaseActive = true` and allow vertical scrolling within `.showcase__vertical-container`. When the user scrolls back left past the threshold, set `isShowcaseActive = false` and resume horizontal scrolling.

## 2. Typography System

### Font Family

Use a single editorial sans-serif family with three weights. Choose one of these (in order of preference):

1. **Söhne** by Klim Type Foundry (if licensed)
2. **Neue Haas Grotesk** (if licensed)
3. **Inter** (free, Google Fonts fallback)

Three weights required:
- **Book / Regular (400)** — body text, descriptions
- **Medium (500)** — navigation, secondary headings, breadcrumbs
- **Semibold / Bold (700)** — emphasis, kickers, uppercase callouts

```css

}

}

@font-face {
  font-family: 'FamilyDrama';
  src: url('/fonts/soehne-halbfett.woff2') format('woff2');
  font-weight: 700;
  font-display: swap;
}
```

### Responsive Type Scale

Display headings use `min(vw, vh)` to scale relative to the shorter viewport axis. This is critical for the horizontal layout because it prevents text from clipping on narrow or short screens.

**Utility classes:**

```css
/* Display sizes — scale to shorter axis */
.-t-200 { font-size: min(14.2857vw, 25vh); line-height: 0.85; }
.-t-150 { font-size: min(10.7143vw, 18.75vh); line-height: 0.88; }
.-t-125 { font-size: min(8.9286vw, 15.625vh); line-height: 0.9; }
.-t-100 { font-size: min(7.1429vw, 12.5vh); line-height: 0.92; }
.-t-80  { font-size: min(5.7143vw, 10vh); line-height: 0.95; }
.-t-60  { font-size: min(4.2857vw, 7.5vh); line-height: 1.0; }

/* Body sizes — fixed */
.-t-24  { font-size: 24px; line-height: 1.4; }
.-t-20  { font-size: 20px; line-height: 1.5; }
.-t-18  { font-size: 18px; line-height: 1.55; }
.-t-16  { font-size: 16px; line-height: 1.6; }

/* Mobile overrides */
@media (max-width: 1024px) {
  .-t-200 { font-size: 66px; }
  .-t-150 { font-size: 56px; }
  .-t-125 { font-size: 48px; }
  .-t-100 { font-size: 40px; }
  .-t-80  { font-size: 34px; }
  .-t-60  { font-size: 28px; }
}
```

**Why `min(vw, vh)`:** On a standard 1920×1080 screen, `min(14.2857vw, 25vh)` resolves to `min(274px, 270px)` = 270px. On an ultrawide 2560×1080, it resolves to `min(366px, 270px)` = 270px. The `vh` cap prevents absurdly large text on wide monitors. On a tall portrait monitor, the `vw` value keeps it proportional.

## 5. Animation System

### Global Easing

Every animation and transition on the page uses a single easing curve:

```css
:root {
  --ease: cubic-bezier(0.4, 0, 0.2, 1);
}
```

This curve is slightly aggressive on entry and gentle on exit. It feels confident without being flashy. Do not use `ease`, `ease-in-out`, or `linear` anywhere.

### In-View Reveal (Default Animation)

Elements start invisible and translated down, then animate into position when they enter the viewport (via Intersection Observer in the horizontal scroll context).

```css
.reveal {
  opacity: 0;
  transform: translateY(100px);
  transition: opacity 0.8s var(--ease), transform 0.8s var(--ease);
}

.reveal.is-visible {
  opacity: 1;
  transform: translateY(0);
}
```

**Intersection Observer setup for horizontal scroll:**

Since the page scrolls horizontally, you cannot use a standard Intersection Observer with the viewport as root. Instead, calculate visibility based on each element's position relative to the current horizontal scroll offset.

```javascript
function checkVisibility() {
  const scrollX = getCurrentScrollX();
  const viewportLeft = scrollX;
  const viewportRight = scrollX + window.innerWidth;

  document.querySelectorAll('.reveal').forEach(el => {
    const rect = el.getBoundingClientRect();
    const elLeft = rect.left + scrollX;
    const elRight = elLeft + rect.width;

    // Element is in viewport when any part overlaps
    if (elRight > viewportLeft && elLeft < viewportRight) {
      el.classList.add('is-visible');
    }
  });
}

// Call on every frame of the horizontal scroll animation
window.addEventListener('horizontalscroll', checkVisibility);
```

### Cascade / Stagger Animation

For lists and sequential items, each child gets an increasing transition delay:

```css
/* Generate stagger delays for up to 20 children */
.cascade > *:nth-child(1)  { transition-delay: 0ms; }
.cascade > *:nth-child(2)  { transition-delay: 50ms; }
.cascade > *:nth-child(3)  { transition-delay: 100ms; }
.cascade > *:nth-child(4)  { transition-delay: 150ms; }
.cascade > *:nth-child(5)  { transition-delay: 200ms; }
.cascade > *:nth-child(6)  { transition-delay: 250ms; }
.cascade > *:nth-child(7)  { transition-delay: 300ms; }
.cascade > *:nth-child(8)  { transition-delay: 350ms; }
.cascade > *:nth-child(9)  { transition-delay: 400ms; }
.cascade > *:nth-child(10) { transition-delay: 450ms; }
/* Continue to nth-child(20) at 50ms increments */
```

Each child also needs the `.reveal` base class so it starts hidden and animates in with the stagger timing.

### Parallax

Parallax is driven by a CSS custom property `--parallax-value` that gets updated via JavaScript based on the element's position relative to the viewport center.

```css
.parallax-element {
  transform: translateY(calc(var(--parallax-value, 0) * 1px));
  will-change: transform;
}
```

```javascript
function updateParallax() {
  const scrollX = getCurrentScrollX();
  const viewportCenter = scrollX + window.innerWidth / 2;

  document.querySelectorAll('[data-parallax]').forEach(el => {
    const rect = el.getBoundingClientRect();
    const elCenter = rect.left + scrollX + rect.width / 2;
    const distance = viewportCenter - elCenter;
    const speed = parseFloat(el.dataset.parallax) || 0.1;
    el.style.setProperty('--parallax-value', distance * speed);
  });
}
```

Usage in HTML: `<div class="parallax-element" data-parallax="0.15">...</div>`

The `data-parallax` value controls speed. `0.05` = subtle drift. `0.2` = pronounced movement. Most elements should use `0.08` to `0.15`.

### Clip-Path Reveals

Some images use clip-path polygon transitions to reveal:

```css
.clip-reveal {
  clip-path: polygon(0 100%, 100% 100%, 100% 100%, 0 100%);
  transition: clip-path 1s var(--ease);
}

.clip-reveal.is-visible {
  clip-path: polygon(0 0, 100% 0, 100% 100%, 0 100%);
}
```

This reveals the image from bottom to top. For left-to-right reveals, adjust the polygon coordinates accordingly.

### Slide-In From Off-Screen

Elements that enter from off-screen (images, text blocks):

```css

}

}

.slide-in-left.is-visible {
  opacity: 1;
  transform: translateX(0);
}
```

## 6. Video Component Pattern

Every video on the page follows the same pattern: a poster image loads first, then a muted autoplay video fades in over the poster when the element enters the viewport.

```html
<div class="media-container">
  <img
    class="media-image"
    src="/assets/poster-image.jpg"
    alt=""
    loading="lazy"
  />
  <video
    class="media-video"
    src="/assets/video-clip.mp4"
    muted
    loop
    playsinline
    disablepictureinpicture
    preload="none"
    poster="/assets/poster-image.jpg"
  ></video>
</div>
```

```css
.media-container {
  position: relative;
  width: 100%;
  height: 100%;
  overflow: hidden;
}

.media-image {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  object-fit: cover;
  z-index: 1;
  transition: opacity 0.6s var(--ease);
}

.media-video {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  object-fit: cover;
  z-index: 0;
  opacity: 0;
  transition: opacity 0.6s var(--ease);
}

/* When video is ready and playing, crossfade */
.media-container.video-active .media-image {
  opacity: 0;
}

.media-container.video-active .media-video {
  opacity: 1;
}
```

**JavaScript video activation:**

```javascript
function initVideoComponents() {
  const containers = document.querySelectorAll('.media-container');

  containers.forEach(container => {
    const video = container.querySelector('.media-video');
    if (!video) return;

    // Only start playing when element is in viewport
    const observer = new IntersectionObserver((entries) => {
      entries.forEach(entry => {
        if (entry.isIntersecting) {
          video.play().then(() => {
            container.classList.add('video-active');
          }).catch(() => {
            // Autoplay failed (browser policy), keep poster visible
          });
        } else {
          video.pause();
          container.classList.remove('video-active');
        }
      });
    }, { threshold: 0.3 });

    observer.observe(container);
  });
}
```

## 7. Slide-by-Slide Build Specs

### SLIDE 1 — Hero / Identity Statement

**Purpose:** First impression. Tell the visitor who Family Drama is and what they do in under 3 seconds.

**Layout:**

```
┌─────────────────────────────────────────────────┐
│                                                 │
│  [breadcrumb: about]          [logo]   [Menu]   │
│                                                 │
│                                                 │
│         FAMILY DRAMA is fluent in…              │
│         ▸ Commercials          ← cycling word   │
│                                                 │
│                                                 │
│                        ┌────────────────────┐   │
│                        │                    │   │
│                        │   HERO IMAGE       │   │
│                        │   (slides in from  │   │
│                        │    off-screen      │   │
│                        │    right)          │   │
│                        │                    │   │
│                        └────────────────────┘   │
│                                                 │
└─────────────────────────────────────────────────┘
```

**Hero image:** Starts at `transform: translateX(100%)` and slides to `translateX(0)` on page load (not on scroll, on initial load). Use a 1.2s transition with the global easing. The inner image is scaled 1.05x to allow subtle parallax movement as the user starts scrolling.

**Text positioning:** The line "FAMILY DRAMA is fluent in…" is `position: absolute; top: 42%; left: 8vw;`. "FAMILY DRAMA" is in the display heading class (.-t-80). "is fluent in…" is the same size but font-weight 400 (book).

**Word cycling mechanic:**

Below "is fluent in…" there is a container holding a stack of words, positioned absolutely. Only one word is visible at a time. As the user scrolls horizontally, the words swap out with a vertical translate animation.

```html
<div class="hero-cycling">
  <div class="hero-cycling__track">
    <span class="hero-cycling__word" data-index="0">Commercials</span>
    <span class="hero-cycling__word" data-index="1">Branded Content</span>
    <span class="hero-cycling__word" data-index="2">Documentaries</span>
    <span class="hero-cycling__word" data-index="3">Director Driven Storytelling</span>
    <span class="hero-cycling__word" data-index="4">Special Projects</span>
    <span class="hero-cycling__word" data-index="5">Making It Personal</span>
  </div>
</div>
```

```css
.hero-cycling {
  position: relative;
  overflow: hidden;
  height: 1.2em; /* Match one line height */
}

.hero-cycling__track {
  display: flex;
  flex-direction: column;
  transition: transform 0.4s var(--ease);
}

.hero-cycling__word {
  height: 1.2em;
  display: flex;
  align-items: center;
  font-size: inherit;
  font-weight: 400;
  white-space: nowrap;
}
```

**Cycling logic:** Divide slide 1's scroll range into 6 equal segments. As `scrollX` passes each threshold, update `transform: translateY(-${index * 1.2}em)` on `.hero-cycling__track`. The final word "Making It Personal" should align with the exit point of slide 1.

## 8. Mobile Responsive Behavior (≤ 1024px)

On screens 1024px wide or narrower, the entire horizontal scroll system is disabled. The page becomes a standard vertically scrolling document.

### Key changes:

```css
@media (max-width: 1024px) {
  .page-about {
    position: relative; /* Not fixed */
    height: auto; /* Not 100vh */
    overflow-y: auto; /* Allow normal vertical scroll */
    overflow-x: hidden;
  }

  .page-about__wrapper {
    flex-direction: column; /* Stack slides vertically */
    width: 100%;
    height: auto;
  }

  .slide {
    width: 100%;
    height: auto;
    min-height: 100vh; /* Each slide is at least full viewport height */
    padding: 80px 24px; /* Horizontal padding for text */
  }

  /* Fixed-position elements become relative */
  .hero-text-fixed {
    position: relative;
    top: auto;
  }

  /* Display type scales down to fixed pixel values */
  .-t-200 { font-size: 66px; }
  .-t-150 { font-size: 56px; }
  .-t-125 { font-size: 48px; }
  .-t-100 { font-size: 40px; }

  /* Sequential reveal items show all at once */
  .cascade > * {
    transition-delay: 0ms !important;
  }

  /* Hidden/transparent text becomes visible */
  .tr-context {
    color: var(--text-primary);
  }

  .tr-kicker {
    opacity: 1;
    transform: translateY(0);
  }

  /* Absolutely positioned images become relative stacked blocks */
  .slide img[style*="position: absolute"] {
    position: relative !important;
    width: 100% !important;
    height: auto !important;
    margin-bottom: 32px;
  }

  /* Showcase nested scroll becomes normal vertical stack */
  .showcase__vertical-container {
    height: auto;
  }

  .showcase__card {
    height: 80vh; /* Slightly shorter on mobile */
  }
}
```

### Mobile navigation:

The menu panel takes `width: 100vw` on mobile (full screen, not 90vw). The main content scale effect still works but scales to 20% instead of 33%.

- Hero images: max 1920px wide, compressed to <200KB
- Thumbnail/secondary images: max 1200px wide, compressed to <120KB
- All images use `loading="lazy"` except the slide 1 hero (which loads eagerly)

### Video Optimization
- All videos compressed to H.264 MP4, max 8MB per clip
- Videos use `preload="none"` and only load when their slide approaches the viewport
- Poster images load first (JPEG, <100KB each)
- Maximum 2 videos playing simultaneously (pause distant videos)

### Animation Performance
- All animations use `transform` and `opacity` only (GPU-composited properties)
- No animations on `width`, `height`, `top`, `left`, `margin`, or `padding`
- `will-change: transform` on the main wrapper and any parallax elements
- Remove `will-change` from elements that have finished animating (via JavaScript)

### Loading Strategy
- Critical CSS inlined in `<head>`
- Fonts preloaded: `<link rel="preload" href="/fonts/..." as="font" crossorigin>`
- Slides 1 through 3 load assets eagerly, all others load lazily as they approach
